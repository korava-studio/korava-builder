# KORAVA Builder AI Rules

1. Treat AI-generated code as advisory: review before committing.
2. Keep AI assistance aligned with existing architectural patterns.
3. Use AI to generate small, testable units instead of monolithic code.
4. Avoid overfitting to a single prompt: prefer reusable abstractions.
5. Document AI-driven changes clearly in code comments or docs.
6. Maintain the same coding standards for AI-assisted implementations.
