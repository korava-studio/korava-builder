# KORAVA Builder Coding Rules

1. Keep files small and focused.
2. Prefer single-responsibility modules and classes.
3. Use meaningful, expressive names for functions, classes, and variables.
4. Avoid duplicated logic by extracting shared behavior into services or utilities.
5. Favor dependency injection over hard-coded module loading.
6. Use strict TypeScript types and avoid `any`.
7. Validate inputs and handle failures gracefully.
8. Write code that is easy to read before optimizing for performance.
